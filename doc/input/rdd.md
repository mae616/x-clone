# 🧑‍💻 ユースケース
## コンテキスト（AI用）
- シナリオ1: ユーザー一覧を見て、気になるユーザーをフォローする
- シナリオ2: 文章を投稿し、タイムラインでフォロー中ユーザーの投稿を閲覧する
- シナリオ3: ユーザー名をクリックしてプロフィール画面に遷移し、そのユーザーの投稿一覧を確認する
- MVP範囲: ユーザー一覧、フォロー/アンフォロー、文章投稿、タイムライン、プロフィール画面

## 人間レビュー用
- **ユーザー一覧** → フォロー/アンフォロー → **タイムラインで投稿閲覧** → **プロフィール確認**
- 認証なし（viewerId固定）でシンプルに動作確認できる構成

# 🏗️ 機能要件
## コンテキスト（AI用）
- 機能一覧（優先度順）:
  1. ユーザー一覧表示
  2. フォロー/アンフォロー（トグル）
  3. 文章投稿（Firestore保存＋即時表示）
  4. タイムライン表示（フォロー中 + 自分の投稿、createdAt降順）
  5. プロフィール画面（ユーザー情報＋投稿一覧）
  6. フォロー数/フォロワー数の集計表示
  7. ユーザー名からプロフィール画面への遷移（タイムライン/ユーザー一覧）

- DoD（完了条件）:
  - ユーザー一覧表示 → 5人以上表示される
  - フォロー/アンフォロー → トグル動作し、UIとDBが一致
  - 文章投稿 → Firestore保存され即時表示
  - タイムライン → フォロー中 + viewerのユーザーのみ、createdAt降順
  - プロフィール画面 → ユーザー情報＋その投稿一覧
  - フォロー数/フォロワー数 → Firestoreから正しく集計
  - タイムライン/ユーザー一覧 → ユーザー名からユーザー画面に遷移できる

# ⚙️ 技術スタック
## コンテキスト（AI用）
- 言語/ランタイム: TypeScript + Node.js
- フレームワーク: React (latest) + Vite
- スタイリング: Tailwind CSS
- ルーティング: react-router-dom
- 状態管理: useState / useEffect（グローバル状態管理なし）
- DB/インフラ: Cloud Firestore（Firebase Emulator Suite）
- 静的解析/整形: ESLint（TypeScript対応、recommended設定）+ Prettier
- テスト戦略: Playwright CLI（結合テスト / E2E）
- 観測性: なし（ローカル環境）

## 注意事項（AI用）
- グローバル状態管理は入れない（Redux等は使わない）
- viewerIdはログインの代替。UIにユーザー切替を入れるならstate管理にする
- 自分自身（viewerId）にはフォローボタンを表示しない
- APIレスポンスをそのままstateに入れ、派生計算は極力バックエンド側に寄せる

## 人間レビュー用
- Firebase Emulatorによりローカル完結。外部サービスへの依存なし
- Vite + React + TypeScriptの標準的な構成
